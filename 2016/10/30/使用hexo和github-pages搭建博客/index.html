使用GitHub和GitHub Pages搭建博客

0. GitHub Pages 和 Hexo

GitHub Pages是GitHub推出的一款免费的静态网站发布服务。网站的文件部署在GitHub repository (直译是仓库，实质上是一个储存空间) 上，GitHub用户可以通过GitHub Pages建立个人或项目的网页。如果你对英文比较了解，GitHub官网上的教程简单明了，可以点击查看。

GitHub支持使用纯HTML来编辑页面。同时，也支持静态网站生成器，也可以说是一种架构来帮助你建静态网站。Jekyll和Hexo是两种非常流行的工具。通过他们，用户可以简单地使用Markdown发布博客。而且，这两种工具都有开源的主题可以选择。

GitHub Pages 官方教程使用Jekyll作为静态网站生成器。但是，我使用的是Hexo作为我的静态网页生成工具。

1.2 GitHub 的设置

1.2.1 为网站声明一块空间

首先，是要为网站建立一个repository去储存网站的文件。对于个人或者组织层级的网站，可能享用username.github.io 这样格式的域名去建立的话，需要建立的repository(储存代码空间)也必须命名为username.github.io。 比如小红的GitHub用户名是xiaohong，小红要建立个人主页希望可以通过xiaohong.github.io去访问自己的页面。那么小红新建了一个repository名字就叫做xiaohong.github.io。当然假若小红并不在意域名，想用自己的项目作为页面的网址比如xiaohong.github.io/hello。就是小红使用她的GitHub中一个名为hello的repository建立的网站。
